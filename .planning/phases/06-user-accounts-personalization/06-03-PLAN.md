   - At the top of the server component, check session via `auth.api.getSession({ headers: await headers() })` — this is optional/non-blocking (page works without auth)

If the existing `getFeedItems` only accepts a single topic string, filter to the first saved topic and show a note like "Showing: Healthcare" with a "Show all" link.
     c. Batch-fetch which items are bookmarked via `getBookmarkedIds(session.user.id)`, create a Set
   - Pass `showBookmark={!!session}` and `isBookmarked={bookmarkedIds.has(item.id)}` to each FeedItemCard
   - If user has no saved topics (or is unauthenticated), show the default unfiltered feed as today

   - Same pattern: check session, if exists check isBookmarked for this item
   - Render BookmarkButton on the detail page if user is authenticated
   - Place it near the title or in the header area of the detail view
