## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Resolved npm peer dependency conflict during better-auth install**
- **Found during:** Task 1
- **Issue:** better-auth's optional SvelteKit dependency pulls `vite@8` which conflicts with the project's `vitest@4` (which uses `vite@7`)
- **Fix:** Installed with `--legacy-peer-deps`; the conflict is in an optional peer dependency for SvelteKit which is not used in this Next.js project
- **Files modified:** package.json, package-lock.json

**2. [Rule 1 - Bug] Used db:push instead of db:migrate for applying new schema**
- **Found during:** Task 1
- **Issue:** `drizzle-kit migrate` failed with "relation system_status already exists" — the migration journal was out of sync with the actual DB state (first migration 0000 already applied but journal not tracking it)
- **Fix:** Used `npm run db:push` which compares current schema against live DB state and applies only the delta (new auth and personalization tables)
- **Files modified:** none (DB-only change)

**3. [Rule 1 - Bug] Fixed getUserTopics test mock chain mismatch**
- **Found during:** Task 2 test execution
- **Issue:** The shared `makeSelectChain` helper returned `{ orderBy, limit }` from the where mock, but `getUserTopics` awaits `.where()` directly (no orderBy/limit in that query)
- **Fix:** Wrote inline mocks for the two `getUserTopics` tests that match the actual query chain shape
- **Files modified:** src/features/auth/queries.test.ts
